# 10mm_socket_programmer_utilities
Templates, scripts, build configs, accumulated cheatsheet references.

It will likely include arbitrary specific placeholders instead of properly formatted placeholders in much of the notes. I'm primarily using this for myself but will format cleaner examples for general use when time permits! 

Because this is primarily for myself, I likely won't accept excessive files or folders of content not relevant to what I am doing. 


Typically I have a particular notation style for text documents. It was built out of either laziness or MD files lacking navigability in some terminals. I'll compare it to MD with my notation on the line below each MD line if possible.:
```
/// Headers

#
//////

##
/////

###
////

####
///

#####
//

The most significant header is dynamic. Sometimes /// Would denote the "largest" header if there are only two levels. There is no max. ///////////// is valid if needed. 

/// Special headers
/// Title ///

/// File start

***
***
***

/// Comments

//
Same as smallest header, basically an HTML p tag.

//{}
Multiline comment.

//{
  Like
  this.
}

//{
  Or
  this
  for
  clarity.
}//

Would be scoped if it was for anything more than text files. 

/// Code block (if using one would clarify anything)
//C{
  // hi
}C//
```
